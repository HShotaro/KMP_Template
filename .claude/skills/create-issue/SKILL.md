# create-issue スキル

GitHub issue を作成してリポジトリにアップロードする。

## 使い方

```
/create-issue <issue の概要>
```

例:
- `/create-issue HomeScreenで楽曲一覧が空になるバグ`
- `/create-issue 検索画面にフィルター機能を追加したい`
- `/create-issue` （引数なしで対話的に作成）

---

## 実行手順

### ステップ 1: 情報収集

`$ARGUMENTS` が空の場合は以下を確認する:
1. issue の種別（バグ報告 / 機能要望 / その他）
2. タイトル（何が起きているか / 何をしたいか）
3. 詳細説明

`$ARGUMENTS` がある場合は、その内容を元に issue 種別を自動判定し、必要に応じて補足情報をユーザーに確認する。

**issue 種別の判定基準:**
- 「バグ」「エラー」「クラッシュ」「動かない」「表示されない」→ `bug`
- 「追加」「機能」「対応」「できるようにしたい」→ `enhancement`
- それ以外 → ユーザーに確認

### ステップ 2: issue 本文を生成する

種別に応じたテンプレートで本文を生成する。テンプレートは `.claude/skills/create-issue/template.md` を参照。

### ステップ 3: ユーザー確認

生成した issue のタイトル・本文・ラベルをユーザーに提示し、承認を得る。

修正が必要な場合は修正してから再確認する。

### ステップ 4: GitHub にアップロード

承認後、以下のコマンドで issue を作成する（`<OWNER>/<REPO>` はプロジェクトのリポジトリに合わせて変更する）:

```bash
gh issue create \
  --title "<タイトル>" \
  --body "<本文>" \
  --label "<ラベル>" \
  --repo <OWNER>/<REPO>
```

作成後、issue の URL をユーザーに返す。

---

## ラベル

| 種別 | ラベル |
|------|--------|
| バグ報告 | `bug` |
| 機能要望 | `enhancement` |
| ドキュメント | `documentation` |
| その他 | （ラベルなし） |

## 注意事項

issue の作成は外部への公開操作のため、内容を提示してユーザーの最終確認を取ってから `gh issue create` を実行する。
