# commit スキル

ステージング済みの変更を解析し、コミットメッセージを生成してコミットする。

## 実行手順

1. `git diff --staged` で変更内容を確認する
2. 変更の種別・意図からコミットメッセージを生成する
3. メッセージを提示してユーザーに確認を求める
4. 承認後に `git commit -m "..."` を実行する

ステージング済みの変更がない場合は「ステージされた変更がありません。`git add` してください」と伝える。

## コミットメッセージ規約

```
<type>: <概要>（日本語、50文字以内）

<本文>（必要な場合のみ）
```

| type | 用途 |
|------|------|
| `feat` | 新機能の追加 |
| `fix` | バグ修正 |
| `refactor` | 動作を変えないリファクタリング |
| `test` | テストの追加・修正 |
| `docs` | ドキュメント・コメントの変更 |
| `chore` | ビルド設定・依存関係・CI の変更 |
| `style` | フォーマット・命名などコードスタイルの変更 |

- 概要は日本語で命令形（「〜を追加」「〜を修正」）
- 本文は「なぜ」変更したかを書く（「何を」は diff から自明なため不要）
- 1コミット1変更単位（複数の独立した変更は分割を提案する）
