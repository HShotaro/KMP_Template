# review スキル

選択中のコードまたは現在の変更差分をレビューする。ルール準拠チェックとコードレビュー（品質・正確性）を一括して実施する。

> **使い分け**: ユーザーが `/review` で手動実行する用。
> ワークフロー（`/add-feature` など）のフェーズ6では、コンテキスト汚染防止のため `reviewer` エージェント（サブエージェント）を使うこと。

## 実行手順

1. `.claude/skills/review/checklist.md` を Read してレビュー観点を確認する
2. `git diff HEAD` で変更内容を確認する
3. 変更されたファイルを Read して checklist.md の観点を順に検証する
4. 問題を重要度別に整理して報告する

## 出力フォーマット

問題があれば以下の形式で報告する:

```
[重要度: 高/中/低] ファイル名:行番号
問題の説明
修正案（あれば）
```

問題がなければ「レビュー完了: 問題なし」と報告する。
